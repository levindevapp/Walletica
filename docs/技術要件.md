# 支出分析アプリ 技術要件書（Ver.0.1）

# 1. 基本方針

## 開発目的

本アプリは個人利用を前提とした支出分析Webアプリケーションとして開発する。

GitHubにはポートフォリオとしてソースコードを公開するが、アプリ自体を常時公開・運用することは現時点では想定しない。

将来的に完成度が高まった場合は、一般公開も検討する。

---

## 開発コンセプト

* 個人利用に最適化する
* ランニングコストを発生させない
* 保守しやすい構成にする
* 将来的な機能追加に対応できる設計とする
* ポートフォリオとしても評価される技術構成を採用する

---

| 項目     | 採用技術        |
| ------ | ----------- |
| データベース | SQLite      |
| ORM    | Drizzle ORM |

SQLiteを採用する理由

* 個人利用では十分な性能を持つ
* データベースサーバーの構築が不要
* 月額費用が発生しない
* DBファイル単位でバックアップできる
* セットアップが容易
* 将来的にPostgreSQL等への移行がしやすい

# 11. ルーティング

## 採用技術

初期開発では **React Router** を採用する。

## 採用理由

* Reactでの採用実績が多い
* 情報量が多く、調査しやすい
* 画面遷移中心の一般的なWebアプリに向いている
* 学習コストが低い
* 本アプリでは複雑な型安全ルーティングまでは不要
* ポートフォリオとしても十分伝わる

## ルーティング方針

* ページ単位でルートを定義する
* 一覧画面の検索条件はURLクエリで管理する
* 月次・年次分析の対象年月もURLクエリで管理する
* ブラウザバックで検索条件・分析条件が保持されるようにする

## 想定ルート

| パス                  | 画面      |
| ------------------- | ------- |
| `/`                 | ダッシュボード |
| `/expenses/new`     | 支出入力    |
| `/expenses`         | 支出一覧    |
| `/analysis/monthly` | 月次分析    |
| `/analysis/yearly`  | 年次分析    |
| `/categories`       | カテゴリ管理  |
| `/settings`         | 設定      |

## URLクエリ例

```text
/expenses?month=2026-07&category=food&payment=credit-card
/analysis/monthly?month=2026-07
/analysis/yearly?year=2026
```
